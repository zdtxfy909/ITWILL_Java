/* 주석문(여러줄 주석처리)
■ 명명규칙(반드시 지킬 것 - 컴파일오류 발생) - 한글은 사용하지 말자
 - 영문대소문자, 숫자, 특수문자(_, $) 사용 가능(단, $는 사용자제할 것)
 - 대소문자 구분사용, 길이제한 없음
 - 예약어(keyword) 단독 사용 안됨
 - 숫자로 시작 안됨 (Ex01 허용, 01EX 안됨)
 (******* 주의 : 한글은 사용하지 말것 **********)

명명규칙(개발자들간의 약속)
■ 패키지명 - package
	전체를 소문자로 작성하고 폴더(디렉토리)구분자는 점(.) 사용
	예) java.lang, java.util, com.oracle.net
	도메인을 역순으로 적용한 후 서비스명, 모듈명 등을 추가해서 작성
	예) 도메인명 mystduy.com
	패키지명 적용 com.mystudy.작성모듈명 
	
■ 클래스(class), 인터페이스(interface)
 - 명사를 주로 사용(권장사항)
 - ★클래스(인터페이스)명의 첫글자는 대문자로 시작 한다.(Hello, String, System ...)
 - ★클래스(인터페이스)명칭이 복합단어인 경우 첫글자를 대문자로 시작한다.
 - (주의) 특수문자 달러기호($), 언더바(_) 사용은 피한다
	예) HelloWorld, StringBuilder, StringBuffer, ....
	
■ 변수명(variable), 메소드명(method)
 - 변수명, 메소드명의 첫글자는 소문자로 한다. (person, num, data ...)
 - 변수명, 메소드명이 복합단어인 경우 복합단어의 첫글자를 대문자로 시작한다.
 	ex) personName, totalValue, computeTotal(), getPersonInfo() ...)
 - 변수명의 길이는 사용범위에 따라 적절하게 사용
   (좁은 경우 짧게, 넓은 경우 충분한 의미전달 가능하게 사용)
 - 메소드명의 시작은 동사로 시작하길 권장함
 
■ 상수화된 변수명 : 상수(constant)
 - 상수명은 전체 대문자로 작성하고 복합어 구분은 언더바(_) 사용
	ex) PI, MIN_VALUE, MAX_VALUE, ...
